Plant Monitoring System

CS 3307 - Group 01 - March 2026

A real-time plant soil moisture monitoring system with serial sensor input, data filtering, calibration, and multiple output channels.

Optional add-on: simple plant photo matching (no model training) for:

  • aloe
  • pothos

Also supports healthy vs unhealthy checking per species using your own reference photos.

Build

mkdir build && cd build
cmake .. && make

Run Demo

# Terminal 1: Virtual serial port
socat -d -d pty,raw,echo=0,link=/tmp/plant_serial_write pty,raw,echo=0,link=/tmp/plant_serial_read
# Terminal 2: Mock sensor
python3 mock_arduino.py /tmp/plant_serial_write
# Terminal 3: Application (update config.hpp to use /tmp/plant_serial_read, then rebuild)
cd build && ./plant_monitor

Verify

  • Console shows readings every 10 seconds
  • Alert message appears when soil becomes dry
  • Recovery message appears when soil becomes healthy
  • Web interface: http://localhost:8080
  • Database: sqlite3 plant_monitor.db "SELECT * FROM readings LIMIT 5;"

Features

  • Serial port communication with Arduino sensor
  • Moving average filter (5-sample window)
  • ADC to moisture percentage calibration
  • Dry soil detection (3 consecutive readings below threshold)
  • SQLite database for persistent storage
  • Console UI with formatted output
  • Web dashboard on port 8080
  • Discord webhook notifications
  • GPIO LED alerts (Raspberry Pi)
  • Multi-threaded (web server runs separately)
  • Graceful shutdown on Ctrl+C

Architecture

Sensor → SensorReader → Filter → Calibration → WaterDecider

Console + Web + Database + Discord + LED

Edit include/config.hpp:

namespace config {
inline constexpr const char *SERIAL_PORT = "/dev/ttyACM0"; // or /tmp/plant_serial_read for demo
inline constexpr int BAUD_RATE = 9600;
inline constexpr int DRY_VALUE = 590; // ADC value in open air
inline constexpr int WET_VALUE = 273; // ADC value in water
inline constexpr double MOISTURE_THRESHOLD = 66.2;
inline constexpr int CONSECUTIVE_DRY_COUNT = 3;
inline constexpr int SAMPLING_INTERVAL_S = 10;
}

Simple Plant Matching (No Training)

Put your own reference photos into folders:

vision/reference/aloe/
vision/reference/pothos/

Add at least 5-10 clear photos per class for better matching.

Predict from a photo file:

source .venv/bin/activate
python3 vision/plant_matcher.py predict-image --image /path/to/query.jpg --json

Predict from MacBook camera:

source .venv/bin/activate
python3 vision/plant_matcher.py predict-camera --camera 0 --json

If camera index 0 fails, try 1.

Healthy vs Unhealthy Check (No Training)

Use one species folder per plant type and include health words in filenames.

Example folder and filenames:

vision/reference/aloe/aloe_healthy_1.jpg
vision/reference/aloe/aloe_yellow_1.jpg
vision/reference/pothos/pothos_healthy_1.jpg
vision/reference/pothos/pothos_unhealthy_1.jpg

Filename mapping used by the script:

  • healthy, good, normal, fresh -> healthy
  • unhealthy, rot, rotting, yellow, wilt, diseased, sick, brown -> unhealthy

Interactive flow (asks species, then asks camera vs image path):

source .venv/bin/activate
python3 vision/plant_matcher.py check-health

Direct camera command:

python3 vision/plant_matcher.py predict-health-camera --species aloe --camera 0 --json

Direct image command:

python3 vision/plant_matcher.py predict-health-image --species pothos --image /path/to/query.jpg --json

Key Components

  • SensorReader: Serial port communication
  • MovingAverageFilter: Noise reduction
  • Calibration: ADC to percentage conversion
  • WaterDecider: Dry soil detection logic
  • Database: SQLite persistence
  • ConsoleUI: Terminal output
  • StatusWeb: HTTP server (port 8080)
  • DiscordNotifier: Webhook alerts
  • LedAlert: GPIO control

How It Works

  1. Reads sensor data from serial port (format: RAW:xxx)
  2. Applies 5-sample moving average filter
  3. Converts ADC value (0-1023) to moisture percentage (0-100)
  4. Tracks consecutive dry readings
  5. Triggers alert after 3 consecutive readings below 30%
  6. Outputs to multiple channels simultaneously
  7. Stores all readings in database

Demo Sequence

The mock Arduino simulates:

  1. Healthy soil (60-70% moisture)
  2. Gradual drying (40-50%)
  3. Becomes dry (<30%, alert triggers)
  4. Critical dry (5-10%)
  5. Watering event (recovery)
  6. Back to healthy

Troubleshooting

Missing SQLite3:

brew install sqlite3 # macOS
sudo apt-get install libsqlite3-dev # Ubuntu

Can't open serial port:

  • For demo: Update config.hpp to /tmp/plant_serial_read
  • For real hardware: Check Arduino connection and permissions

Port 8080 in use:

  • Change port in main.cpp: web.start(8081);

Missing socat:

brew install socat # macOS
sudo apt-get install socat # Ubuntu
